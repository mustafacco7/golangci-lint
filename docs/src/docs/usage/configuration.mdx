The config file has lower priority than command-line options. If the same bool/string/int option is provided on the command-line
and in the config file, the option from command-line will be used.
Slice options (e.g. list of enabled/disabled linters) are combined from the command-line and config file.

To see a list of linters enabled by your configuration use:

```sh
golangci-lint linters
```

## Config File

GolangCI-Lint looks for config files in the following paths from the current working directory:

- `.golangci.yml`
- `.golangci.yaml`
- `.golangci.toml`
- `.golangci.json`

GolangCI-Lint also searches for config files in all directories from the directory of the first analyzed path up to the root.
If no configuration file has been found, GolangCI-Lint will try to find one in your home directory.
To see which config file is being used and where it was sourced from run golangci-lint with `-v` option.

Config options inside the file are identical to command-line options.
You can configure specific linters' options only within the config file (not the command-line).

There is a [`.golangci.reference.yml`](https://github.com/golangci/golangci-lint/blob/master/.golangci.reference.yml) file with all supported options, their description, and default values.
This file is neither a working example nor a recommended configuration, it's just a reference to display all the configuration options.

The configuration file can be validated with the JSON Schema: https://golangci-lint.run/jsonschema/golangci.jsonschema.json

## Configuration File Properties

The configuration file for `golangci-lint` includes the following properties:

- `version`: The version of `golangci-lint` to use. This is a required field.
- `name`: The name of the custom binary. This is an optional field with a default value of `custom-gcl`.
- `destination`: The directory path used to store the custom binary. This is an optional field with a default value of `.`.
- `plugins`: A list of plugins to integrate inside the custom binary. Each plugin can be specified with the following properties:
  - `module`: The module name of the plugin.
  - `import`: The import path to use for the plugin.
  - `version`: The version of the module. This is required for modules available through a Go proxy.
  - `path`: The path to the local module. This is required for local modules.
  - `description`: A description of the plugin.

## Examples

Here are some examples of how to use the configuration file:

### Example 1: Basic Configuration

```yaml
version: v1.56.2
name: custom-golangci-lint
destination: ./my/path/
plugins:
  - module: 'github.com/example/plugin3'
    version: v1.2.3
  - module: 'github.com/example/plugin4'
    import: 'github.com/example/plugin4/foo'
    version: v1.0.0
  - module: 'github.com/example/plugin2'
    path: /my/local/path/plugin2
  - module: 'github.com/example/plugin1'
    path: ./my/local/path/plugin1
  - module: 'github.com/example/plugin2'
    import: 'github.com/example/plugin4/foo'
    path: /my/local/path/plugin2
```

### Example 2: Configuration with Descriptions

```yaml
version: v1.56.2
name: custom-golangci-lint
destination: ./my/path/
description: This is a custom configuration for golangci-lint.
plugins:
  - module: 'github.com/example/plugin3'
    version: v1.2.3
    description: This is a plugin from a Go proxy.
  - module: 'github.com/example/plugin4'
    import: 'github.com/example/plugin4/foo'
    version: v1.0.0
    description: This is a plugin from a Go proxy with a specific import path.
  - module: 'github.com/example/plugin2'
    path: /my/local/path/plugin2
    description: This is a plugin from a local source with an absolute path.
  - module: 'github.com/example/plugin1'
    path: ./my/local/path/plugin1
    description: This is a plugin from a local source with a relative path.
  - module: 'github.com/example/plugin2'
    import: 'github.com/example/plugin4/foo'
    path: /my/local/path/plugin2
    description: This is a plugin from a local source with an absolute path and a specific import path.
```

## Command-Line Options

```sh
golangci-lint run -h
{.RunHelpText}
```

When the `--cpu-profile-path` or `--mem-profile-path` arguments are specified, `golangci-lint` writes runtime profiling data
in the format expected by the [pprof](https://github.com/google/pprof) visualization tool.

When the `--trace-path` argument is specified, `golangci-lint` writes runtime tracing data in the format expected by
the `go tool trace` command and visualization tool.

## Cache

GolangCI-Lint stores its cache in the subdirectory `golangci-lint` inside the [default user cache directory](https://pkg.go.dev/os#UserCacheDir).

You can override the default cache directory with the environment variable `GOLANGCI_LINT_CACHE`; the path must be absolute.
